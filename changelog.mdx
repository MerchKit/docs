<Update label="July 21, 2026" description="API v1 redesign: one shape everywhere, labeled references, upsert & batch, sources, and a consolidated MCP surface">

  The public API and MCP server were redesigned ahead of general availability. **This is a breaking
  release** — the beta had no external integrators, so no compatibility shims were kept. The five
  developer guides were rewritten to match; start with the new
  [first five minutes](/developers/overview).

  **New**
  - **One shape everywhere.** Every entity — list row, detail, and mutation response — is the same
    [one-map resource](/developers/conventions#resource-anatomy): system fields (`id`, `type`,
    `label`, `parent_id`, timestamps) at the top level, every customer-defined key inside a sparse
    `attributes` map. Mutations return the full resource, so no follow-up `GET` is ever needed.
  - **References carry labels inline.** A reference reads as `{id, type, label}` — no N+1 name
    lookups — and list references arrive as complete ordered arrays. Writes are symmetric: a read
    stub re-sent verbatim is a valid write.
  - **Upsert and batch, by your own key.** `POST /{resource}/upsert` and `POST /{resource}/batch`
    (≤100 items, per-item statuses) create-or-update by any text attribute you name — a 5,000-SKU
    ERP delta push is now 50 idempotent calls with no client-side ID map.
  - **Sources are a first-class resource.** `POST /v1/sources` schedules the scrape-and-create
    pipeline (`202` + job to poll); attaching to a product is a `data_sources` reference write.
  - **Shorter filters.** Bare `filter[sku]=…` means equals; operators are now `eq`, `ne`,
    `contains`, `not_contains`, `starts_with`, `ends_with`, `gt`/`gte`/`lt`/`lte`, `exists`
    (replacing `blank`/`notBlank`), and a new `in` for batch lookups (≤200 values).
  - **An [errors page](/developers/errors)** documents every error code and field-level issue code;
    `documentation_url` in error responses now resolves there.

  **Changed (breaking)**
  - `/assets*` is now **`/images*`**; `/{id}/history` is now `/{id}/events`; `/{id}/attributes` is
    gone — `PATCH /{id}` is the one write path; `grid-views` and `attribute-classes` left public
    REST (they remain on MCP).
  - snake_case everywhere: write bodies, MCP arguments, and responses all agree
    (`parent_entity` → `parent_id`, `{entityId, entityType}` → `{id, type}`).
  - **Scopes are one taxonomy** across REST and MCP: a symmetric read/write/delete triple per
    resource, images and sources included. `write:attribute_values`, `read:completeness`, and
    `read:history` are gone; issued keys were
    [remapped automatically](/developers/authentication#scopes).
  - **MCP tools consolidated to generic tools** — `list_entities`, `get_entity`, `create_entity`,
    `update_entity`, `upsert_entity`, and friends, with `type` as a parameter. The per-type tools
    (`search_products`, `get_product`, `set_attribute_values`, `set_parent`, `get_history`) retired;
    shapes are identical to REST.

</Update>

<Update label="Week of July 13, 2026" description="MCP tools for vendors, categories, and images">

  **New**
  - **Work with vendors, categories, and images from an agent.** Two new MCP tools — `list_entities` and `create_entity` — let Claude and other agents look up or create the entities your products reference, then link them by ID. Ask your agent to "file these shoes under Trail Running and set the vendor to Salomon" and it can now do the whole thing: find or create the category and vendor, then set the references. See [working with vendors, categories, and images](/developers/mcp#working-with-vendors-categories-and-images).
  - **Enrich more than products.** `set_attribute_values` now accepts an `entity_type`, so an agent can update an image's alt text, fix a vendor's name, or edit a category — not just products.

  **Fixed**
  - Asking the API for attribute definitions with an unknown entity type returns a clear validation error instead of an opaque failure.

</Update>

<Update label="Week of June 29, 2026" description="Wayfair sandbox, smarter CSV imports, and Shopify image sync polish">

  **New**
  - **Wayfair sandbox credentials.** Connect a Wayfair integration with sandbox keys, run a full mapping and export end to end, then swap in production credentials when you're ready. See [adding a channel](/product/channels-and-content-generation/adding-a-channel).
  - **Merge attribute on CSV import.** Pick which attribute Merchkit should match incoming rows against, and apply a filter so an import only touches the subset of products you intend to update. Useful for partial vendor refreshes and targeted price or inventory loads. See [reimporting and updating products](/product/sources-and-importing/reimporting-updating-products).
  - **Child entities in product list references.** Variants and other child products can now be selected directly in a product-list reference cell, not just their parents.

  **Updated**
  - Wayfair class views are labeled with the class ID, and a **Refresh classes** button resets the view to the latest taxonomy from Wayfair.
  - Filters in the catalog grid handle inverse, parent, and other reference attributes, and the operator list is scoped to the ones that make sense for each filter type.
  - The **Usage** page is paginated, so workspaces with long event histories load quickly.
  - Default grid pagination size is larger, so you scroll less between pages on big catalogs.
  - The Lulu print-on-demand payload was updated to match the latest field list.
  - File names are preserved when sources are uploaded, so the original filename shows up against each import job.

  **Fixed**
  - **Shopify image sync** is more reliable: newly created Shopify images are polled so their filenames sync back, previously unlinked images get relinked on the next sync, and image format and extension are saved correctly on import.
  - Entity references can be assigned in import and export mappings again.
  - The image reference editor no longer mishandles deletions while you're searching, and child products can pull an image directly from their parent.
  - The OpenAPI spec served to the docs API playground uses an absolute server URL, so **Try it** requests resolve correctly. See the [API overview](/developers/overview).

</Update>

<Update label="June 2026" description="Wayfair, the API and MCP server, the AI agent, and roles & permissions">

  **Channels**
  - **Sell on Wayfair, end to end.** Map your catalog to Wayfair's product taxonomy, auto-classify products into the right categories, and validate values against Wayfair's accepted attributes before you export. Connect **sandbox** credentials to test the full flow, then switch to production when you're ready.
  - A single workspace can now carry multiple attribute classes, so distinct channel taxonomies live side by side.

  **Developers (beta)**
  - **Customer API and MCP server.** Operate your catalog programmatically over a REST API at `/api/v1` (OpenAPI 3.1), or connect an agent like Claude through the remote MCP server. Available in beta — [contact support](mailto:support@merchkit.com) to enable it for your workspace. See the [API overview](/developers/overview) to get started.
  - API key management is rolling out gradually behind an account flag.

  **AI agent**
  - **Chat with your catalog.** A side-panel assistant answers questions, builds attributes, creates and edits products, and runs bulk changes. It reads the rows and cells you have selected, and asks for your approval before it writes anything.

  **Teams**
  - **Roles and permissions** (gradual rollout). Give teammates Admin, Editor, or Viewer access, and fine-tune what each role can do per workspace.
  - Invitations now use a 6-digit code, so corporate email scanners no longer consume the link before your teammate clicks it.

  **Images**
  - Manage every product image in one full-screen library. Search, preview, upload by file or URL, reorder, and link or unlink images across products or from a parent down to its variants.
  - More reliable two-way image sync with Shopify, including media-name handling and Media ID mapping fixes.

  **Enrichment**
  - Reference lookups now feed generation, so the AI can pull values from linked vendors and products.
  - Large bulk runs process faster and stay within platform limits.

</Update>

<Update label="May 2026" description="Confidence filtering, usage transparency, audit logging, and presence">

  **Enrichment**
  - **Confidence filter.** Every generated value carries a confidence score. Filter the grid to surface low-confidence values for review, or find products that don't have a score yet.
  - Generation now supports JSON responses.
  - Toggling AI generation on or off now preserves your attribute configuration.

  **Usage and billing**
  - A new **Usage** page shows your SKU enrichment allowance and agent credits with progress bars, overage warnings, and per-user attribution. See what you've used before the bill arrives.

  **Channels**
  - Readable integration errors: a failed export now names the field or metafield at fault in plain language.
  - Shopify import runs in resumable batches for large stores.
  - Wayfair attributes are disambiguated by class name.

  **Teams**
  - **Audit logging** records every product create, update, and delete.
  - **Presence indicators** show which teammates are in the workspace and what they're viewing.

  **Workspace**
  - Bulk-refresh URL sources from the sources page.
  - Map imported records to a parent product on update, and trigger relationship mapping yourself.

</Update>

<Update label="April 2026" description="Custom classification, a rebuilt enrichment engine, and reworked CSV export">

  **Enrichment**
  - **Custom classification.** Auto-classify products down a deep, multi-level taxonomy (1,500+ categories) while staying inside your own category tree, with no invented categories.
  - A rebuilt attribute-generation engine delivers faster, more reliable enrichment, with cleaner handling of empty and enum values.

  **Workspace**
  - Column width and pinning now persist and survive import and export.
  - A new custom cell editor replaces the previous grid editor for smoother editing.
  - Collections inherit in both directions across products, data sources, and images.

  **Images**
  - Image generation moved to the gpt-image-2 model.

  **Channels**
  - CSV export now honors your exact selection, including select-all with exceptions.

</Update>

<Update label="March 2026" description="Reference lookups in enrichment, infinite Shopify import, and reusable templates">

  **Enrichment**
  - Generation can read **reference lookups**, pulling values from linked products and vendors. Formulas can use them too.
  - A new specialized data type adds purpose-built editors for structured channel fields.

  **Channels**
  - **Infinite Shopify import** handles stores of any size without timing out.
  - Save and reuse named **import and export mapping templates** per integration.
  - Merchkit now detects and maps Shopify metafields automatically, including unstructured ones.

  **Images**
  - Reorder images in the image collection editor by drag-and-drop.

  **Workspace**
  - Import filters gained an "in list" operator, and primary-ID matching is now whitespace- and case-tolerant, so updates land on the right products instead of creating duplicates.

</Update>

<Update label="February 2026" description="Formula attributes, product collections, and a simpler channel setup">

  **Enrichment**
  - **Formula attributes.** Derive a value deterministically from your other columns using spreadsheet-style functions. Formulas recompute automatically on bulk update and re-import. See the [formula functions reference](/product/reference/formula-functions).

  **Workspace**
  - Group products into **collections**, add variants from the product tree, and import variant structures from CSV.
  - Keyboard shortcuts for column search, and clearer single- and multi-select cells.
  - Choose whether a CSV import is allowed to create new products.

  **Channels**
  - A simpler channel template system, and one-click Shopify connect with automatic cleanup on uninstall.

  **Images**
  - A faster default image-generation model, with no model picker to manage.

</Update>

<Update label="January 2026" description="Shopify on GraphQL, CSV import onboarding, and read-only attributes">

  **Channels**
  - Shopify export moved to the **GraphQL API**, with variant-level export for SKUs, prices, barcodes, and options.
  - Automatic detection when the Merchkit app is installed, plus updated webhooks.
  - Build the structured JSON payload Lulu print-on-demand needs straight from your attributes.

  **Workspace**
  - A guided **CSV import onboarding** flow.
  - **Read-only attributes** protect source-of-truth fields from manual edits, while still accepting imports, syncs, and formula writes.
  - Export errors now write back to the originating grid cell.
  - Search now matches the attribute names you see on screen.

  **Enrichment**
  - Point individual attributes at the model that suits them (OpenAI, Anthropic Claude, or Google Gemini).
  - Image extraction retries automatically when a source returns nothing.

</Update>
